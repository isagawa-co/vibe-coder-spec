# /on-failure

Handle build or test failures. Explain in plain English, fix, learn.

## Instructions

1. **Read the failure output**

2. **Explain in plain English:**
   - What broke (no jargon)
   - Why it broke
   - What the fix is

3. **Get user approval for fix approach**

4. **Implement fix**

5. **Re-run tests**

6. **Invoke `/kernel/learn`** to record the lesson
